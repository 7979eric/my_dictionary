# 📚 논문 리딩용 단어장 (HTML 단일 파일)

브라우저에서 실행되는 **로컬 단어장**입니다.
LLM 딸깍으로 제작되었으며, 누구나 손쉽게 만들고 사용할 수 있습니다.
파일 분리는 귀찮아서 하지 않았습니다.(꼬이는게 두렵기도 합니다.)
의학 용어 / 일반 영단어를 분리 관리하고, 검색·정렬·백업/불러오기 기능을 제공합니다.  
데이터는 **브라우저 localStorage**에 저장됩니다(서버/DB 없음).
다른 환경에서 사용하고 싶을 때, 단어 백업 후 로드 하시면 간단히 사용하실 수 있습니다.

단어 뜻이나 이런 거는 그냥 LLM한테 형식에 맞춰서 달라고 한 다음에 복붙하시면 됩니다. API 쓰면 돈나가니깐 그런 거 안넣었습니다.

---

## ✅ 주요 기능

- **카테고리 탭**: 🏥 의학 용어 / 🍏 일반 영단어
- **단어 등록 항목**
  - 단어(원형)
  - 발음기호(IPA) (예: `/ˈtɛmpərəl/`)
  - 뜻
  - 파생어(예: `n. prediction, adj. predictive`)
  - 사용 예시(자동 강조 + `*별표*` 강조 지원)
  - 보충 설명/메모(의학 용어 탭에서만 표시)
- **검색**
  - 검색어가 없을 때: 현재 탭(의학/일반)만 표시
  - 검색어가 있을 때: **의학/일반 전체에서 함께 검색 결과 표시**
  - 검색 범위 선택: `전체 검색 / 단어만 검색`
- **정렬**
  - 최신순(기본) / 이름순(A-Z)
- **편집/삭제**
- **백업/불러오기(JSON)**
  - 불러오기 시 **중복 단어 자동 건너뜀**(카테고리+단어 기준, 대소문자 무시)

---

## 🚀 실행 방법

### 1) 그냥 실행(간단)
- 파일 더블클릭으로 브라우저에서 열기  
  (대부분 동작하지만, 일부 브라우저/보안 정책에 따라 동작 차이가 있을 수 있습니다.)

### 2) VS Code에서 Live로 실행(추천)
1. VS Code 확장 `Live Server` 설치
2. HTML 파일 우클릭 → **Open with Live Server**
3. 저장하면 브라우저가 자동 새로고침됩니다.

---

## 💾 데이터 저장 방식

- 단어 데이터는 브라우저 **localStorage**에 저장됩니다.
- 사용 키: `studyVocabData`

> 브라우저를 바꾸거나, 시크릿 모드/캐시 삭제/사이트 데이터 삭제를 하면 데이터가 사라질 수 있습니다.  
> 중요한 데이터는 `📤 백업하기`로 꼭 JSON 저장을 권장합니다.

---

## 📤 백업 / 📥 불러오기

- **백업하기(저장)**: 현재 데이터가 `vocab_backup_YYYYMMDD_HHMMSS.json`로 다운로드됩니다.
- **불러오기**: JSON 파일을 선택해 데이터를 합칩니다.
  - 같은 카테고리에서 **단어가 동일**하면 중복으로 판단하고 추가하지 않습니다.

---

## 🔎 검색 팁

- “단어만 검색”은 `단어/파생어/발음기호`를 대상으로 검색합니다.
- “전체 검색”은 `단어/뜻/파생어/발음기호/예시/보충설명`까지 모두 검색합니다.
- 예시 문장에서 **단어 자동 강조** + `*별표*`로 원하는 구문을 수동 강조할 수 있습니다.

    "derivative": "adj. temporal",

  }
